# M2Apps v1.0.4

## Added

- Native OS service integration for daemon lifecycle management:
  - Linux `systemd`
  - Windows Service (`sc`)
  - macOS `launchd`
- New daemon service commands:
  - `m2apps daemon uninstall`
  - `m2apps daemon enable`
  - `m2apps daemon disable`

## Changed

- Daemon CLI service flow now uses OS service manager abstraction for:
  - `install`, `start`, `stop`, `status`
- Standardized CLI operational output pattern for service actions:
  - `[INFO] ...`
  - `[OK] ...`
  - `[ERROR] ...`
- Service manager contract expanded with:
  - `Uninstall()`
  - `Enable()`
  - `Disable()`

## Security & Safety

- Linux service operations now require root privileges.
- Windows service operations now require Administrator privileges.
- Binary path validation added before service installation.
- Service installation is blocked when service already exists (no overwrite).

## Version

- Application version bumped to `v1.0.4`.
