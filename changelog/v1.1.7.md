# M2Apps v1.1.7

## Fixed

- Improved Windows daemon start resilience when elevated service start fails, with automatic process-based fallback attempt.
- Added explicit daemon manual fallback guidance after start failures:
  - `m2apps daemon run`
  - `m2apps daemon run --detach`

## Improved

- Enhanced `m2apps daemon run` UX:
  - always prints ASCII banner at startup
  - prints current version in run header
  - supports detached/background mode via `--detach` / `-d`
  - supports foreground run without stream tailing via `--no-log`
- Split daemon logging into dedicated files:
  - service/runtime log: `daemon.service.log`
  - Local API access log: `daemon.access.log`
- Enhanced log streaming UI:
  - separate source labels for service and API logs
  - severity-aware colorized output for warning/error/failed entries
  - stream starts from current EOF so only active runtime logs are shown (no old history mixed in).
- Added Local API access logging in daemon runtime with method, path, status, and duration metadata.

## Version

- Application version bumped to `v1.1.7`.
