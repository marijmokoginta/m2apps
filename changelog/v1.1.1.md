# M2Apps v1.1.1

## Fixed

- Fixed Linux installer elevated relaunch path issue in `install.sh` that caused:
  - `bash: ./install.sh: no such file or directory`
- Installer now resolves and uses absolute script path before privilege escalation.

## Improved

- Enhanced `m2apps list` output into a structured table format.
- Added detailed app metadata in list output:
  - app id
  - name
  - version
  - channel
  - preset
  - install path
- Added running status information per app in list output:
  - `stopped`
  - `running (x/y)`
  - `unknown` when process status cannot be resolved

## Version

- Application version bumped to `v1.1.1`.
