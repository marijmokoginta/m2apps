# M2Apps v1.1.3

## Fixed

- Fixed self-update flow where final update/error information could disappear too quickly because the interactive screen returned to main menu immediately.
- Fixed self-update behavior on Unix-like systems to ensure binary replacement completes cleanly without forcing immediate automatic menu transition.

## Improved

- Self-update now keeps final logs visible until user confirmation before returning to main menu.
- Self-update error handling now pauses and waits for user input, so failure details remain readable.
- On successful Unix self-update, CLI now informs user to restart m2apps to use the newly installed binary.
- Windows self-update restart path now also waits for explicit user confirmation before closing the current process.

## Version

- Application version bumped to `v1.1.3`.
