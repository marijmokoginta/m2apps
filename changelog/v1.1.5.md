# M2Apps v1.1.5

## Fixed

- Fixed Local API diagnostics false-negative caused by authorization-protected probe endpoints (`401`) by adding unauthenticated health endpoints for daemon checks.
- Fixed daemon runtime ownership and permission issues that caused `permission denied` when reading `daemon.port`/`daemon.pid` in mixed root/non-root flows.
- Fixed stale daemon runtime behavior by detecting unreachable API behind existing PID and forcing a clean restart path.
- Fixed self-update and dev install binary-lock conflict (`file is busy`/`text file busy`) by stopping daemon service before binary replacement and restoring it after update.

## Improved

- Added Local API public diagnostic endpoints:
  - `GET /health`
  - `GET /ping`
- Updated daemon interactive diagnostics to use the new health/ping endpoints.
- Added app env synchronization on daemon startup so `M2APPS_API_URL` (and related API vars) are re-injected when daemon port changes.
- Added cross-platform base directory override via `M2APPS_HOME` and propagated it through service/elevation flows.
- Improved Linux service unit generation:
  - uses resolved current executable path
  - injects `M2APPS_HOME`
  - runs service under resolved non-root runtime user/group when available
  - performs ownership migration for existing runtime files/directories.
- Improved macOS and Windows service installation to use resolved executable path (instead of hardcoded binary path assumptions).
- Added shared runtime resolver utility for service executable path handling.
- Enhanced `scripts/dev.sh` to auto-stop/start daemon around binary copy to avoid lock conflicts during local development.
- Extended internal self-update helper with daemon restart control (`--restart-daemon`) for coordinated service recovery after update.

## Version

- Application version bumped to `v1.1.5`.
