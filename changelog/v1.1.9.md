# M2Apps v1.1.9

## Highlights
- Introduced Requirement Install Engine (Phase 15) for guided dependency handling in `m2apps install`.
- Added configurable requirement install mode (`assisted` and `manual`).
- Added Windows-specific notice to restart terminal after dependency install when PATH/env changes.

## Changes
- Added Phase 15 implementation document:
  - `docs/phases/phase-15-requirement-install.md`
- Requirement strategy context document added:
  - `docs/architecture/m2apps-requirement-strategy.md`
- Config and validation:
  - Added `install_mode` in install config schema.
  - Validation now supports `assisted` (default) and `manual`.
- Requirement result enrichment:
  - Added missing metadata (`Missing`, `Reason`) with reasons: `not_found`, `version_mismatch`, `unknown`.
- New guided requirement installation package:
  - `internal/reqinstall/plan`
  - `internal/reqinstall/resolver`
  - `internal/reqinstall/executor`
  - `internal/reqinstall/manual`
  - `internal/reqinstall/flow.go`
- Install flow integration:
  - `cmd/install.go` now executes `check -> missing -> plan -> confirm -> install -> re-check` before app install continues.
- Version target handling enhancement:
  - Tool target version is now derived from requirement constraint (not hardcoded tool-version mapping).

## Version
- Application version bumped to `v1.1.9`.
