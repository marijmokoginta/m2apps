# M2Apps v1.1.0

## Added

- Supervisor elevation trigger for privileged daemon operations:
  - Linux: PolicyKit popup (`pkexec`)
  - Windows: UAC popup (`RunAs`)
- Self-update engine for M2Apps binary with:
  - latest release check via GitHub API
  - update prompt in interactive startup flow
  - skip-until-next-version state in `~/.m2apps/self_update.json`
- Internal updater command path for Windows locked-binary replacement:
  - `m2apps internal self-update`
- Daemon runtime log writer:
  - `~/.m2apps/logs/daemon.log`

## Improved

- Daemon process spawn now uses OS-specific detached attributes for reliability.
- Daemon startup now validates listener port binding and returns explicit errors when port is in use.
- Daemon run flow auto-starts installed apps (`AutoStart=true`) and avoids duplicate process starts.
- Installer privilege behavior now triggers OS auth popup for machine-level install targets.

## Metadata

- App metadata now supports `auto_start` with default compatibility behavior for existing configs.

## Version

- Application version bumped to `v1.1.0`.
