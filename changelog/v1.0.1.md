# M2Apps v1.0.1

## Scope

This changelog includes all v1.0.1 changes, including the previous commit (`ebc9705`) and the latest enhancement updates.

## Added

- Interactive root CLI menu with keyboard navigation (`↑`, `↓`, `Enter`, `Esc`).
- Main menu flows for:
  - install application
  - update installed application (interactive app selection)
  - switch update channel (interactive app + channel selection)
  - delete installed application (interactive app selection)
- New command:
  - `m2apps delete <app_id>`
- Cross-platform installer scripts:
  - `install.sh` (Linux/macOS)
  - `install.ps1` (Windows)
- Release notes automation in GitHub Actions using `changelog/<tag>.md`.

## Changed

- Root command displays explicit version `v1.0.1`.
- `m2apps list` now reads installed apps from metadata storage and outputs detected app IDs/names.
- Installed app discovery now falls back to folder-based detection if app metadata cannot be parsed.
- Daemon manager supports app registry removal via `UnregisterApp(appID)` for delete flow.
- Linux/macOS installer now downloads archive assets (`.tar.gz`) from GitHub Release and extracts before install.
- Windows installer now downloads archive asset (`.zip`) from GitHub Release and extracts before install.
- Release packaging script now includes installer scripts in distribution structure.
- README includes OS-specific M2Apps storage path documentation.

## Fixed

- App list detection issue where existing app folders were not shown in some metadata failure scenarios.
- Installer asset resolution mismatch (binary vs archive naming) against release asset naming in workflow.

## Release Assets

- `m2apps-windows-amd64.zip`
- `m2apps-linux-amd64.tar.gz`
- `m2apps-darwin-amd64.tar.gz`

## Compatibility

- Existing commands remain available:
  - `install`
  - `update`
  - `channel`
  - `daemon`
  - `list`
