# M2Apps v1.2.1

## Highlights
- Improved Windows daemon start reliability by hardening elevation detection and adding elevated action status handshake.
- Replaced generic "privileged operation failed with exit code 1" behavior with actionable child-process error reporting.

## Changes
- Windows privilege detection:
  - Strengthened elevated/admin detection with layered checks:
    - PowerShell Administrator role check
    - `fltmc` fallback
    - `net session` fallback
- Windows service admin check:
  - `internal/service/windows.go` now uses shared privilege checker (`privilege.IsElevated()`) for consistency.
- Daemon elevated action handshake:
  - Added internal command:
    - `m2apps internal daemon-action --action <...> --status-file <...>`
  - Parent daemon command now launches elevated internal action and waits for status-file result.
  - Child action returns structured success/failure status with detailed error messages.
- Improved daemon command supervision flow:
  - Elevated daemon actions now return explicit failure details instead of generic exit-code-only errors.

## Version
- Application version bumped to `v1.2.1`.
