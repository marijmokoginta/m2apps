# M2Apps v1.1.4

## Fixed

- Fixed self-update failure on protected system paths (`/usr/local/bin`) caused by insufficient write permission during binary backup/replace.
- Self-update now detects permission-denied errors during binary replacement and triggers elevated internal helper flow automatically.

## Improved

- Added supervisor-elevated fallback path for self-update binary replacement without forcing immediate restart on Unix-like systems.
- Extended internal self-update helper with explicit restart control (`--restart`) to support controlled post-update UX flow.
- Preserved final self-update completion logs while still applying privileged replacement when needed.

## Version

- Application version bumped to `v1.1.4`.
