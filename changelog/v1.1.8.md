# M2Apps v1.1.8

## Highlights
- Added Laravel/Laravel-Inertia server mode support to run app web server on:
  - `localhost` (`127.0.0.1`)
  - `lan` (`0.0.0.0` with local device IPv4 injection to `APP_URL`)
- Added smarter cleanup flow for install and update to remove transient artifacts after successful move/apply.

## Changes
- Config + validation:
  - Added `server_mode` in `install.json` schema.
  - Validation now accepts `localhost|lan` and limits `lan` mode to `laravel`/`laravel-inertia` presets.
- Storage metadata:
  - Added `server_mode` to encrypted app metadata.
  - Backward-compatible default to `localhost` when field is missing.
- Runtime process behavior:
  - Process manager now resolves runtime host and `APP_URL` host based on preset + `server_mode`.
  - Added local IPv4 detection utility for LAN mode.
  - Improved process URL rendering to show reachable LAN IP when host is `0.0.0.0`.
- Preset/runtime support:
  - Added host-mode helper module.
  - Integrated smart preset handling and app URL change hooks.
- Cleanup enhancements:
  - `install`: remove `install.json`, downloaded archive, and `.m2apps_tmp` after successful install flow.
  - `update`: remove downloaded update package and app temp `.m2apps_tmp` after successful apply.

## Version
- Application version bumped to `v1.1.8`.
