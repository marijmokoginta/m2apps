# M2Apps v1.2.0

## Highlights
- Fixed Windows self-update false-success behavior by waiting for updater helper status before marking update as successful.
- Hardened daemon stop and app process stop/restart flows to avoid false stopped/running states.
- Improved requirement installer fallback strategy when package manager targets are unavailable.

## Changes
- Self-update (Windows):
  - Added updater helper status handshake file (`success`/`fail`) and timeout handling.
  - Parent process now validates helper result before printing update success.
  - Added elevated relaunch fallback in internal helper when binary replace hits permission error.
- Daemon stop strictness:
  - `daemon stop` now verifies Local API is no longer reachable before returning success.
  - Daemon manager stop now performs stricter PID termination and runtime file cleanup (`daemon.pid` and `daemon.port`).
- App process stop/restart hardening:
  - Improved stop error propagation and detection for permission-related failure signals.
  - Added stricter stop wait/verification behavior.
  - Linux process liveness detection improved for zombie/PID-reuse edge cases.
  - Added elevation retry fallback for stop/restart actions.
- Linux privilege behavior:
  - Elevation now falls back to `sudo` when `pkexec` is unavailable/fails.
- Requirement installer fallback:
  - Added additional fallback commands using `curl` references for Linux/Windows when package targets are missing.
  - Added post-install per-tool re-check to prevent false `[OK]` status.

## Compatibility Notes
- Daemon and app auto-start behavior on reboot remains unchanged:
  - daemon service auto-start still managed by OS service manager,
  - app auto-start still managed by daemon startup flow for apps with `AutoStart=true`.

## Version
- Application version bumped to `v1.2.0`.
